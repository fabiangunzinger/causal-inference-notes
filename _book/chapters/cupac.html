8  CUPAC

  • CUPED was designed by folks at Doordash (Tang et al. 2000) to increase the precision of their switchback experiments.

  • CUPED can be extended to (make notation consistent with CUPED)

\[ y' = y - \theta \bar{f(X)} + \theta E(f(X)) \]

  • In above, it can be shown that optimal \(f(X) = E(Y|X)\) (show this).

  • CUPED, which is effectively regression adjustment, uses best linear predictor.

  • CUPAC extends this to non-linear predictors, generating \(\hat{y} = g(X) = E(Y|X)\).

8.1 Thoughts

  • There are two approaches: predict the experiment-time outcome or predict the pre-experiment value.

  • If you do the former, then you need features that are unaffected by the treatment. This means that for reach metric and each experiment, you need to collect features that meet this requirement. This will be different for each metric and experiment because whether a feature is impacted by the treatment depends on the specific experiment (i.e. where in the funnel the feature is active). Hence, building this into a pipeline in an automated way would be an enormous endeavour, if it’s possible at all.

  • Hence, this approach is really only viable in a very specialised setting where you keep running the same type of experiment, and thus have to build and train the model manually and only once (this is how it’s being used for courier experiments)
